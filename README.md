# Telegram Bot для управления клиентами VPN в панели 3x-ui
## Основные возможности:
- **Создание нового клиента**
- **Интегрированная база данных SQLite** для учета пользователей, фиксации постоянного сопоставления Telegram ID и Email, а также отслеживания оплаты
- **Подтверждение верификации клиента со стороны админа**
- **Контроль периодов оплаты:**
  - Автоматическая синхронизация дат создания клиентов из базы 3x-ui
  - Категоризация статусов оплаты: Оплачено (0), Истекает (менее 7 дней) (1), Просрочено (2)
  - Фоновая проверка каждые 12 часов с уведомлением администратору о необходимости продления
  - Возможность исключения доверенных пользователей из системы проверки оплат путем добавления их в группу `private` в панели 3x-ui
  - Ручное продление подписок администратором командой `/paid`, либо по запросу пользователя
  - Запуск рассылки администратором по должникам с возможностью отправки кастомного сообщения
- **Безопасность:** Защита от смены пользователем своего @username (все привязки осуществляются по уникальному Telegram ID)
- **Экспорт БД:** Возможность для администратора мгновенно выгрузить файл базы данных SQLite командой `/getdb`

## Команды бота
### Для всех пользователей:
* `/start` — отправка заявки администратору на одобрение доступа
* `/create_token` — генерация личной ссылки на VPN-подписку
* `/help` — просмотр подробной инструкции по установке и настройке клиентов на Windows (v2rayN), Android, iOS и MacOS (V2Box)

### Для администратора:
* `/status` — быстрая проверка статуса оплат пользователей, принудительное обновление БД из панели и запуск рассылки должникам
* `/paid <tg_id> [кол-во месяцев]` — подтверждение оплаты и продление подписки пользователя (по умолчанию на 3 месяца)
* `/getdb` — отправка администратору резервной копии базы данных `users.db`
* `/broadcast <текст>` — отправка глобального сообщения всем пользователям с активным VPN
* `/cancel` — прерывание активных сценариев ввода текста

## Как запустить?

### 1. Переменные окружения (.env)
Создайте файл `.env` в корневой директории бота и заполните его следующими переменными в формате `KEY='VALUE'`:

| Переменная окружения | Описание                                                                                                  |
|----------------------|-----------------------------------------------------------------------------------------------------------|
| BOT_TOKEN            | Токен вашего Telegram-бота                                                                                |
| ADMIN_ID             | Ваш Telegram ID                                                                                           |
| URL                  | URL вашей панели 3x-ui (без слэша на конце)                                                               |
| SUB_URL              | URL сервера подписок вашей панели (без слэша на конце)                                                    |
| API_TOKEN            | API токен (пишется при первой установке панели, можно создать новый в настройки панели -> учётная запись) |
| INBOUND_IDS          | ID протоколов VPN **через запятую без пробелов** для привязки клиентов                                    |

### 2. Создание виртуального окружения и установка зависимостей
```bash
sudo apt install python3-venv -y
python3 -m venv .venv
source .venv/bin/activate
pip install aiogram python-dotenv
```

### 3. Настройка автозапуска бота (Linux)
Для удобства создаём директорию vpn_bot и помещаем туда все файлы бота. 

Затем создаём сервис systemd для автозапуска бота при старте системы:
```bash
sudo vim /etc/systemd/system/vpn-bot.service
```

Заполните файл следующим содержимым:
```text
[Unit]
Description=VPN_BOT_AUTORUN
After=network.target

[Service]
ExecStart=/root/vpn_bot/.venv/bin/python3 /root/vpn_bot/Main.py
WorkingDirectory=/root/vpn_bot/
Restart=always
User=root

[Install]
WantedBy=multi-user.target
```

Обновите конфигурацию systemd и запустите сервис:
```bash
sudo systemctl daemon-reload
sudo systemctl enable vpn-bot
sudo systemctl start vpn-bot
```

Проверить статус сервиса можно командой:
```bash
sudo systemctl status vpn-bot
```

Перезапустить сервис можно командой:
```bash
sudo systemctl restart vpn-bot
```
